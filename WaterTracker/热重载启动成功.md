## 🛠️ 服务器管理

### 使用管理脚本 (推荐)

```bash
cd /workspaces/RUthirsty-cordova/WaterTracker

# 查看状态
./dev-server.sh status

# 停止服务器
./dev-server.sh stop

# 启动服务器
./dev-server.sh start

# 重启服务器
./dev-server.sh restart

# 查看日志
./dev-server.sh logs

# 实时跟踪日志
./dev-server.sh follow
```

### 手动命令

```bash
# 查看进程
ps aux | grep live-server | grep -v grep

# 查看日志
cat /tmp/live-server.log

# 实时日志
tail -f /tmp/live-server.log

# 停止服务器
pkill -f "live-server"

# 启动服务器
cd /workspaces/RUthirsty-cordova/WaterTracker/www
nohup npx live-server --port=8080 --host=0.0.0.0 --no-browser > /tmp/live-server.log 2>&1 &
```

## 🔍 故障排除

### 问题: 浏览器没有自动刷新

**检查**:
1. 服务器是否运行: `./dev-server.sh status`
2. 文件是否保存成功
3. 浏览器控制台是否有错误

### 问题: 端口被占用

**解决**:
```bash
# 停止当前服务器
./dev-server.sh stop

# 修改端口 (编辑 dev-server.sh 中的 PORT 变量)
# 或使用其他端口启动
cd www
nohup npx live-server --port=3000 --host=0.0.0.0 --no-browser > /tmp/live-server.log 2>&1 &
```

### 问题: 服务器崩溃

**解决**:
```bash
# 查看日志
./dev-server.sh logs

# 重启服务器
./dev-server.sh restart
```
